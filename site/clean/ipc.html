IPC - Inter Process Communication

An IPC is a very useful construct, It allows two separate processes running to synchronize between them, and as an extension share data.

As a SW architect this gives the designer freedom in dividing the system into separate functional blocks, each running separately but still sharing their status with the other applications

Linux and C allows for many different solutions to this issue each with it's own advantages and disadvantages, I'll try and record here different options on how these are implemented as well as some code samples for free use.
