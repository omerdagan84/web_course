command return value

each command has a return value. the command value in the code can usually be by the 'exit ' command.
a command success will return '0' value and a failure will return '1' or any non-zero value depending on the running software
one can check the return value by the system variable '?' - by echoing the variable with 'echo $?' one can see the return value of the last command run
for example: test@test:~ $ ps
PID TTY TIME CMD
20748 pts/34 00:00:00 bash
20846 pts/34 00:00:00 ps
test@test:~ $:~ $ ps | grep bush
test@test:~ $:~ $ echo $?
1
test@test:~ $:~ $ ps | grep bash
20748 pts/34 00:00:00 bash
test@test:~ $:~ $ echo $?
0
 as seen in the example if 'grep' was able to find a match it will return '0', if not it will return '1'

command concatenation

COMMAND && COMMAND_TRUE || COMMAND_FALSE for example lets assume one wishes to test if a certain process exists pidof crond > /dev/null && echo "process is running" || echo "process is not running"; let's assume that the process is indeed running the output will be: $ process is running a bit of explanations:
  • pidof - a simple command that displays the pid (process ID) of the processes if it exists
    but we are not interested in the command out put but its return value
  • crond - a cron daemon - a process that usually runs by default and enables one to schedule jobs in many different time definitions "see crond - Linux job scheduler"
  • > /dev/null - since we do not care about the output just the return value we just forward all the output to the 'null' device
  • && - execute command upon return code '0' - which means the following command will be performed upon success
  • || - execute command upon return code other than '0' - which means the following command will be performed upon fail

text file show

the cat command

cat FILENAME the simplest of commands - just displays the file content in ascii representation
NOTE: one shouldn't 'cat' a binary file - since the ascii representation of that file will be incomprehensible cat -n FILENAME will add the line numbers to the print - very useful when combining with other commands

the tail command

tails the file - same as cat but will print the end of the file tail FILENAME #will print the last 10(default) lines of the file
tail -n NUM FILENAME #will print the last NUM lines of the file
tail -f FILENAME #will print the end of the file but will keep on following as it grows the command is especially useful when trying to see changes to a large file, such as logs.
or when trying to review a changing file ('-f' option)

the head command

shows the head of the file - same as tail. lines shown by default is 10. head FILENAME #will show the first 10 lines of the file
head -n NUM FILENAME # will show the first NUM lines of the file

search within files

lets assume one wishes to search source code library for a variable grep -rn STRING LOCATION
grep -rnv STRING LOCATION
  • -r - search recursively within the specified LOCATION
  • -n - show line numbers in the output
  • -v - show only lines NOT matching the pattern string
  • STRING - the STRING to be searched
  • LOCATION - the LOCATION or path to be searched

searching for files

$ ls -d -1 $PWD/*.* | grep animation
/home/dagan/solo_js/animation.html
/home/dagan/solo_js/animation.js
  • ls - list files
  • -d - list the directories themselves
  • -1 - list one file per line
  • $PWD/*.* - list all files with their respective full path
  • | - move command output to the following command
  • grep STRING - grep the files matching the STRING
this allows you to search the current directory files for a matching STRING(file name)
NOTE: one can extend or modify the 'ls' command for more complicated paths - or use 'find'

the awk command

the awk command is a pattern scanning tool, it implements of the AWK programming language and is a very useful tool for processing
though since it is essentially a programming language it is very complicated and extensive knowledge is needed to get the full benefits from it the following example is something I encountered during my work and it shows the power of the command I achieving something that seems quite complicated at first

the exapmles calculates the actual free RAM space in a system, to achieve this one should sum the MemFree and the Cached lines in the Linux meminfo file cat /proc/meminfo | grep -E 'MemFree|^Cached' | awk '{sum += $2} END {print sum}'
  • cat /proc/meminfo - show the content of the meminfo file
  • | - move output to the following command
  • grep -E 'MemFree|^Cached' - use grep with RegEx and get only the MemFree line and the line beginning('^') with Cached
  • | awk - parse the output of the grep command with awk
  • '{sum += $2} END - add the content of the second($2) column in the output line to sum for each line
  • {print sum}' - print the value calculated after parsing all the lines

process management

the ps command

ps aux
ps aux | cat
ps
  • ps - show all processes attached to users ID and attached to the controlling terminal on embedded systems which usually supports a single root terminal this command is enough to show all processes
  • ps aux - show all processes attached to user 'x' which is a wild-card for all users. on some systems one should type ps -aux
  • ps aux | cat - a TIP when running on embedded systems with restricted terminal. sometimes command output is cut after 84charecters - this trick allows one to bypass this restriction

the pidof command

pidof COMMAND_NAME shows the PID of the COMMAND_NAME if it exists (return value '0') or nothing if it doesn't (return value '1')

the kill commands

kill PID
killall COMMAND_NAME
  • kill PID - kill a specific PID
  • killall COMMAND_NAME - kill all instances of a certain program
